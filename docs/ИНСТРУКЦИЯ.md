# Инструкция по применению

Краткое руководство по работе с приложением **Word-шаблоны**: загрузка шаблонов, управление данными (импорт из Excel в БД, выбор записей), заполнение полей и генерация документов.

---

## 1. Запуск приложения

После установки зависимостей и активации виртуального окружения выполните:

```bash
python main.py
```

Откроется главное окно «Word-шаблоны» с панелью шаблонов, таблицей полей и кнопками действий.

---

## 2. Загрузка шаблона

1. Нажмите **«Загрузить шаблон…»**.
2. Выберите файл **.docx**, в котором используются переменные Jinja2, например:
   - `{{ клиент }}`
   - `{{ дата_договора }}`
   - `{{ сумма }}`
3. После выбора файла приложение:
   - проверяет синтаксис шаблона;
   - извлекает имена полей из документа;
   - регистрирует шаблон в базе и добавляет его в список «Шаблон».

Шаблон появится в выпадающем списке **«Шаблон:»**. Имя берётся из имени файла (без расширения).

---

## 3. Выбор шаблона и заполнение полей

1. В списке **«Шаблон:»** выберите нужный шаблон (не «<-не выбран->»).
2. В блоке **«Значения полей шаблона»** отобразится таблица:
   - колонка **«Поле»** — имя переменной из шаблона;
   - колонка **«Значение»** — место для ввода или выбора значения.
3. Заполните значения:
   - **вручную** — введите текст в ячейках или выберите вариант из выпадающего списка (если справочник задан в БД);
   - **из записи набора данных** — если шаблон привязан к набору данных, выберите запись в списке «Запись из набора данных:» (значения подставятся в таблицу, их можно поправить);
   - **из Excel в таблицу** — кнопка «Вставить из Excel…» подставляет значения из файла формата «имя / значение» прямо в таблицу полей (без сохранения в БД).

Можно комбинировать: выбрать запись, затем подправить отдельные поля вручную.

---

## 4. Наборы данных и импорт из Excel в БД

Данные для подстановки в шаблоны можно хранить в **наборах данных** (таблицах в БД): клиенты, договоры, акты и т.д. Один набор данных — это список записей; у каждой записи — набор полей (имя поля → значение).

### 4.1. Импорт из Excel в базу

1. В меню выберите **Сервис → Импорт из Excel в БД…**.
2. В поле **«Набор данных»** выберите существующий набор или нажмите **«Создать новый набор…»** и введите название (например, «Клиенты» или «Договоры»). Код создаётся автоматически из названия.
3. Нажмите **«Выбрать файл .xlsx…»** и укажите файл Excel.
4. В файле **первая строка** — заголовки колонок (они станут именами полей записей). Остальные строки — данные; каждая строка = одна запись.
5. Нажмите **OK** для запуска импорта. В сообщении будет указано количество импортированных записей.

Пример структуры Excel для набора данных (имена полей должны совпадать с полями шаблона, если вы планируете подставлять запись в этот шаблон):

| CLIENT_NAME   | CONTRACT_NUMBER | CONTRACT_DATE | AMOUNT  |
|---------------|-----------------|---------------|---------|
| ООО «Ромашка» | 1               | 2026-03-07    | 150000  |
| ИП Иванов     | 2               | 2026-03-08    | 75000   |

После импорта записи хранятся в БД и их можно выбирать при генерации (если шаблон привязан к этому набору данных).

### 4.2. Привязка шаблона к набору данных

Чтобы в главном окне при выборе шаблона появлялся список записей для подстановки:

1. В меню выберите **Сервис → Привязать шаблон к набору данных…**.
2. Выберите **шаблон** и **набор данных** (или «— не привязывать —», чтобы отвязать).
3. Нажмите **OK**.

После привязки при выборе этого шаблона в блоке «Значения полей шаблона» появится строка **«Запись из набора данных:»** с выпадающим списком. Выбор записи заполнит таблицу полей значениями из выбранной записи; при необходимости их можно отредактировать вручную перед генерацией.

---

## 5. Вставка значений из Excel (без сохранения в БД)

Если нужно один раз подставить данные из файла в таблицу полей (без создания записей в БД):

1. Выберите шаблон в списке «Шаблон:».
2. Нажмите **«Вставить из Excel…»**.
3. Укажите файл **.xlsx** или **.xls**.
4. В файле должны быть две колонки:
   - **имя поля** (заголовок: «имя», «name», «поле» или «field»);
   - **значение** (заголовок: «значение» или «value»).
5. После выбора файла значения подставятся в таблицу по совпадению имён полей (без учёта регистра).

Пример структуры Excel:

| имя           | значение      |
|---------------|---------------|
| CLIENT_NAME   | ООО «Ромашка» |
| CONTRACT_DATE | 07.03.2026    |
| AMOUNT        | 150 000       |

---

## 6. Генерация документа

1. Убедитесь, что выбран шаблон и при необходимости заполнены поля (вручную, из записи или из Excel).
2. Нажмите **«Сгенерировать документ»**.
3. В диалоге сохранения укажите имя и папку для итогового файла **.docx**.
4. После успешной генерации появится сообщение с путём к сохранённому файлу.

В документе все переменные вида `{{ имя_поля }}` будут заменены на значения из таблицы. Факт генерации и снимок контекста сохраняются в истории (БД).

---

## 7. Типичные сценарии

| Задача | Действия |
|--------|----------|
| Новый шаблон | «Загрузить шаблон…» → выбрать .docx → заполнить поля → «Сгенерировать документ». |
| Заполнить вручную | Выбрать шаблон → ввести значения в таблице → «Сгенерировать документ». |
| Заполнить из записи БД | Привязать шаблон к набору данных (Сервис → Привязать…) → импортировать Excel в этот набор (Сервис → Импорт из Excel в БД…) → в главном окне выбрать шаблон → в «Запись из набора данных» выбрать запись → при необходимости поправить поля → «Сгенерировать документ». |
| Один раз из Excel в таблицу | Выбрать шаблон → «Вставить из Excel…» → выбрать .xlsx (формат имя/значение) → «Сгенерировать документ». |
| Ещё один документ по тому же шаблону | Изменить значения в таблице (или выбрать другую запись, или снова «Вставить из Excel…») → «Сгенерировать документ» → указать новое имя файла. |

---

## 8. Настройки

- **Сервис → Настройки…** — вкладки «Интерфейс» (каталог шаблонов по умолчанию, раскрытие панели лога при запуске) и «БД» (тип БД, путь к SQLite или параметры PostgreSQL). Изменения БД применяются после перезапуска приложения.

---

## 9. Ограничения и советы

- **Шаблон**: поддерживаются переменные в формате `{{ переменная }}`. Сложные конструкции Jinja2 (циклы, условия) могут поддерживаться docxtpl, но извлечение полей в приложении ориентировано на простые подстановки.
- **Имена полей**: при импорте из Excel (и в наборах данных) имена полей сопоставляются без учёта регистра с полями шаблона. Для подстановки из записи БД имена полей в наборе данных должны совпадать с именами переменных в шаблоне.
- **База данных**: по умолчанию используется SQLite (`word_templates.db`). В БД хранятся шаблоны, поля, справочники значений полей, **наборы данных и записи**, история генераций.
- При ошибке загрузки шаблона проверьте, что в .docx нет опечаток в `{{ }}` и файл не повреждён.

Подробное описание архитектуры и плана реализации см. в [ARCHITECTURE.md](ARCHITECTURE.md) и [IMPLEMENTATION_PLAN.md](IMPLEMENTATION_PLAN.md).
