Ориентир для пошагового внесения изменений в код с сохранением существующего стиля проекта.

---

## Этап 1 — Подготовка структуры проекта

- **Оставить без переименования**: `main.py`, `main_window.py`, `settings.py`, `settings_dialog.py`, `db/connection.py`, `core/`, `logger/`, `ui/main_window.ui`.
- **Изменить**: `db/models.py` (добавить модели данных и связь шаблона с набором данных), `docs/ARCHITECTURE.md` (уже обновлён).
- **Создать**: `services/data_table_service.py`, `services/excel_import_service.py`, при необходимости `ui/dialogs/` для диалогов (или оставить диалоги в корне/в `main_window`).
- **Пометить как legacy (не удалять)**: текущий сценарий «Вставить из Excel» в главном окне, который только подставляет имя/значение в таблицу без сохранения в БД — оставить как есть, добавить рядом сценарий «Импорт в БД» (в диалоге или отдельной кнопке).

## Этап 2 — Модели БД

- В `db/models.py`:
  - Добавить **DataTable**: id, name, code (unique), description, created_at, updated_at.
  - Добавить **DataTableColumn**: id, data_table_id (FK CASCADE), field_name, display_name, sort_order; UniqueConstraint(data_table_id, field_name).
  - Добавить **DataRecord**: id, data_table_id (FK CASCADE), created_at, updated_at.
  - Добавить **DataRecordValue**: id, data_record_id (FK CASCADE), field_name, value_text; UniqueConstraint(data_record_id, field_name).
  - В **DocumentTemplate** добавить **linked_data_table_id** (Integer, FK to data_tables.id, nullable, ondelete SET NULL).
- Создать связи (relationships) и вызвать `create_all_tables` при старте — новые таблицы появятся в БД.
- Импорт из Excel: сохраняем записи в data_records + data_record_values; при первом импорте колонки можно создавать из заголовков листа.
- Ручное редактирование: CRUD по data_records и data_record_values через data_table_service.

## Этап 3 — Импорт Excel

- **Модуль**: `services/excel_import_service.py`.
- Читать .xlsx через openpyxl (data_only=True), активный лист.
- Первая строка — заголовки; нормализовать имена (пробелы, регистр) в field_name (например, «Имя клиента» → client_name или оставить как есть по соглашению).
- Сопоставление колонок: по точному совпадению field_name с data_table_columns; если колонок у таблицы ещё нет — создать data_table_columns из заголовков первой строки.
- Для каждой строки (со 2-й): создать DataRecord, для каждой ячейки с именем колонки — DataRecordValue(record_id, field_name, value). Пустые строки (все ячейки пустые) пропускать.
- Обработка ошибок: логировать и по возможности возвращать краткий отчёт (сколько строк импортировано, сколько пропущено). Дубликаты в MVP не проверяем — каждая строка = новая запись.
- Вызов из UI: диалог или главное окно — выбор набора данных (data_table), выбор файла .xlsx, вызов сервиса, сообщение о результате.

## Этап 4 — Ручной ввод и редактирование

- **UI**: экран или диалог «Наборы данных»: список data_tables, кнопки «Создать», «Импорт Excel», «Редактировать записи».
- Редактирование записей: для выбранной data_table показать таблицу: строки = data_records, столбцы = data_table_columns (field_name/display_name). Добавить строку, удалить строку, редактировать ячейки. Сохранение — вызов data_table_service (create_record, update_record, delete_record, update_record_values).
- В MVP можно ограничиться одним диалогом «Записи набора данных» с QTableWidget и кнопками Сохранить/Добавить/Удалить.

## Этап 5 — Работа с шаблонами Word

- Без изменений логики: регистрация шаблона, извлечение полей (template_parser), валидация, генерация через docxtpl (document_generator).
- В template_service добавить: получение linked_data_table_id для шаблона; при регистрации шаблона не трогать привязку к data_table (оставлять NULL или задавать в отдельном экране).
- Опционально: диалог «Настройки шаблона» — выбор привязанного набора данных (linked_data_table_id).

## Этап 6 — Выбор данных для генерации

- В главном окне при выборе шаблона с linked_data_table_id: показать комбобокс или список «Запись» (записи выбранного набора данных). При выборе записи — заполнить таблицу полей (Поле / Значение) из data_record_values; пользователь может править значения вручную.
- Альтернатива: «Ручной ввод» vs «Из записи»; при «Из записи» — выбор записи и подстановка. Контекст для генерации всегда собирается из текущего состояния таблицы полей (_collect_context_from_table).
- Итоговый контекст: словарь field_name → value (как сейчас), передаётся в generate_document.

## Этап 7 — MVP

- Работает: регистрация шаблонов, выбор шаблона, таблица полей, ручной ввод в таблице, генерация документа.
- Добавлено: создание набора данных (data_table), импорт Excel в набор данных (записи сохраняются в БД), ручное создание/редактирование записей в наборе данных (один диалог), привязка шаблона к набору данных (linked_data_table_id), выбор записи при генерации с подстановкой в таблицу полей.
- Допустимые ограничения в MVP: один набор данных на шаблон; маппинг полей записи и полей шаблона по совпадению имён (field_name); без истории импортов (таблица imports не обязательна); кнопка «Вставить из Excel» в главном окне остаётся для прямого заполнения таблицы полей без сохранения в БД.
